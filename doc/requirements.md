# 要件定義書 - 献立自動作成アプリ

**作成日**: 2026年4月7日  
**バージョン**: 1.0

---

## 1. プロジェクト概要

### 目的
家族の1週間分の献立を自動生成することで、毎日の食事選びの悩みを解消し、食事準備の負担を減らす。

### ユーザー
- **メインユーザー**: 家族（料理担当者が主に操作）
- **利用シーン**: 週の始めに献立をまとめて決めたいとき

## 3. Phase 1 要件定義

### 3-1. 機能要件

#### アカウント機能

| 優先度 | 機能 |
|---|---|
| Must | メールアドレス＋パスワードで会員登録できる |
| Must | ログイン・ログアウトができる |
| Must | パスワードリセット（メール送信）ができる |
| Must | レシピ・献立データはログインユーザーに紐づく |
| Could | 退会機能 |

#### レシピ機能

| 優先度 | 機能 |
|---|---|
| Must | レシピを登録できる |
| Must | レシピ一覧を表示できる |
| Must | レシピを編集できる |
| Must | レシピを削除できる |
| Must | 料理名でレシピを検索できる |
| Must | カテゴリでレシピを絞り込める |

### 3-2. 非機能要件

| 項目 | 内容 |
|---|---|
| プラットフォーム | Webブラウザ（PC・スマホ対応のレスポンシブ） |
| 認証方式 | メールアドレス＋パスワード |
| 画像登録 | なし（Phase 2 以降で検討） |

### 3-3. 画面一覧

| 画面名 | 概要 |
|---|---|
| 会員登録画面 | メールアドレス・パスワードを入力して登録 |
| ログイン画面 | メールアドレス・パスワードでログイン |

|---|---|---|

| パスワード | string | ハッシュ化して保存 |
| 作成日時 | datetime | |

#### レシピ

| フィールド | 型 | 備考 |
|---|---|---|
| ID | string | 一意識別子 |
| ユーザーID | string | 登録したユーザー |

## 4. Phase 2 以降の機能（将来対応）

### Phase 2 - 献立自動作成 ＋ 家族グループ

#### 献立自動作成

- 献立作成ボタンを押すと1週間分（朝・昼・夜）の献立を自動生成
- 登録済みレシピのデータをもとに生成
- 同じ料理が連続で出ないようにする
- 1食 ＝ ポイント合計が1.0になるように組み合わせる
  - 例：ご飯（主食 0.3）＋ 味噌汁（汁物 0.2）＋ 生姜焼き（主菜 0.5）
- 1品単位での差し替え再生成が可能
- レシピ登録数が不足している場合はエラーを表示
- 生成した献立の保存・閲覧が可能

#### 家族グループ機能

- グループを作成してメンバーを招待できる
- グループ内でレシピ・献立を共有できる

### Phase 3 - 食材・冷蔵庫機能

- レシピに食材を登録できる（食材名・数量・単位）
- 冷蔵庫機能：現在の食材在庫を管理できる
- 料理を「作った」とマークすると、使用した食材分の在庫が自動で減る
- 献立を作成したとき、現在の冷蔵庫在庫と比較して不足食材を表示する
