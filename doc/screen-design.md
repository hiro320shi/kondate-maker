# 画面設計書 - 献立自動作成アプリ Phase 1

**作成日**: 2026年4月12日  
**バージョン**: 1.0

---

## 1. 基本方針

| 項目 | 決定内容 |
|---|---|
| アプローチ | **モバイルファースト** |
| UIライブラリ | **Tailwind CSS** |
| ベースブレークポイント | モバイル（〜639px）をデフォルト設計 |
| タブレット以上 | `sm:` 640px〜 で幅を制限・センタリング |

### レイアウト方針

- **フォーム画面**（登録・ログイン等）: モバイルは全幅 `px-4`、`sm:` 以上は `max-w-sm mx-auto`
- **一覧画面**: モバイルは全幅、`sm:` 以上は `max-w-2xl mx-auto`
- **共通ヘッダー**: 固定（`sticky top-0`）。高さ 56px 目安

### 画面5: レシピ一覧画面 `/recipes`

```
┌─────────────────────────────────┐
│ 🍽 献立メーカー        [ログアウト] │  ← sticky ヘッダー
├─────────────────────────────────┤
│                                 │
│  ┌───────────────────────────┐  │
│  │ 🔍 料理名で検索...         │  │
│  └───────────────────────────┘  │
│                                 │
│  [すべて] [主食] [主菜] [副菜]    │  ← 横スクロール可能なチップ
│  [汁物] [その他]                 │
│                                 │
│  ┌───────────────────────────┐  │
│  │ から揚げ              [主菜] │  │
│  │ ⭐ 0.5pt              ✏️ 🗑  │  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │ 白米                  [主食] │  │
│  │ ⭐ 0.3pt              ✏️ 🗑  │  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │ 味噌汁                [汁物] │  │
│  │ ⭐ 0.2pt              ✏️ 🗑  │  │
│  └───────────────────────────┘  │
│                                 │
│                       ╔═══╗    │
│                       ║ ＋ ║    │  ← FAB（レシピ登録）
│                       ╚═══╝    │
└─────────────────────────────────┘
```

**コンポーネント仕様:**

| 要素 | 詳細 |
|---|---|
| 検索バー | 入力中にリアルタイムでフィルタリング（デバウンス 300ms） |
| カテゴリチップ | 横スクロール可能（`overflow-x-auto flex gap-2`）。選択中は色を変える |
| レシピカード | 料理名・カテゴリバッジ・ポイント・編集/削除ボタン |
| カテゴリバッジ | カテゴリごとに色分け（主食:青、主菜:赤、副菜:緑、汁物:黄、その他:グレー） |
| FAB | 右下固定（`fixed bottom-6 right-6`）。プライマリカラー |
| 削除ボタン | タップで確認ダイアログを表示してから削除 |
| 0件時 | 「まだレシピが登録されていません」+ 登録を促すメッセージ |

**検索・絞り込みの動作:**
- 検索とカテゴリ絞り込みはAND条件
- 「すべて」チップ選択時はカテゴリ絞り込みなし

### 画面6: レシピ登録画面 `/recipes/new`

```
┌─────────────────────────────────┐
│ ← 戻る    レシピを登録           │  ← ヘッダー
├─────────────────────────────────┤
│                                 │
│  料理名 *                        │
│  ┌───────────────────────────┐  │
│  │ 例：から揚げ               │  │
│  └───────────────────────────┘  │
│                                 │
│  カテゴリ *                      │
│  ┌───────────────────────────┐  │
│  │ 選択してください      ▼   │  │
│  └───────────────────────────┘  │
│                                 │
│  ポイント *                      │
│  ┌───────────────────────────┐  │
│  │ 0.5                       │  │
│  └───────────────────────────┘  │
│  0.1〜1.0 / 1食の構成比          │
│                                 │
│  メモ                            │
│  ┌───────────────────────────┐  │
│  │                           │  │
│  │  （任意）                  │  │
│  │                           │  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │         登録する           │  │
│  └───────────────────────────┘  │
│                                 │
└─────────────────────────────────┘
```

**コンポーネント仕様:**

| 要素 | 詳細 |
|---|---|
| 戻るボタン | ヘッダー左。`/recipes` に戻る |
| 料理名 | `type="text"` / 最大100文字 |
| カテゴリ | セレクトボックス。APIから取得した一覧を表示 |
| ポイント | スライダー（`type="range"`）/ `step="0.1"` / `min="0.1"` / `max="1.0"` / 現在値を数値で横表示 |
| ポイントのヒント | 「1食の構成比。合計1.0になる組み合わせで献立を自動作成します」と説明 |
| メモ | `textarea` / 最大500文字 / 高さ 80px |
| 登録ボタン | 全幅、プライマリカラー |

**バリデーション（フロント）:**
- 料理名: 必須
- カテゴリ: 必須
- ポイント: 必須、0.1〜1.0、小数点1桁まで

**成功後:** `/recipes` にリダイレクト + 「レシピを登録しました」のトースト表示

## 5. 共通UI仕様

### カラーパレット（Tailwind）

| 役割 | クラス例 | 用途 |
|---|---|---|
| プライマリ | `bg-emerald-500` | ボタン・FAB・アクセント |
| テキスト | `text-gray-800` | 本文 |
| サブテキスト | `text-gray-500` | ヒント・ラベル |
| 背景 | `bg-gray-50` | ページ背景 |
| カード | `bg-white` | カード背景 |
| ボーダー | `border-gray-200` | 区切り線 |

### カテゴリバッジカラー

| カテゴリ | クラス例 |
|---|---|
| 主食 | `bg-blue-100 text-blue-700` |
| 主菜 | `bg-red-100 text-red-700` |
| 副菜 | `bg-green-100 text-green-700` |
| 汁物 | `bg-yellow-100 text-yellow-700` |
| その他 | `bg-gray-100 text-gray-700` |

### トースト通知

- 画面上部に表示（`fixed top-4`）
- 表示時間: 3秒
- 成功: `bg-emerald-500 text-white`
- エラー: `bg-red-500 text-white`

### 確認ダイアログ（削除時）

```
┌─────────────────────────────────┐
│  レシピを削除しますか？           │
│  この操作は取り消せません。        │
│                                 │
│       [キャンセル]  [削除する]    │
└─────────────────────────────────┘
```
